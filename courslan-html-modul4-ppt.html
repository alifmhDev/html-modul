01 // Komunikasi Web Dua Arah

Sejauh ini, kita membuat web statis (Informasi hanya mengalir satu arah: dari server ke pengguna).

Untuk membangun fitur seperti **Login, Pendaftaran, atau Komentar**, website butuh alat untuk menangkap ketikan pengguna. Di sinilah HTML Forms mengambil alih.

02 // Anatomi Wadah <form>

Tag <form> adalah cangkang (wrapper) wajib untuk semua jenis inputan. Ia memiliki dua atribut krusial yang menentukan *ke mana* dan *bagaimana* data dikirim.

<form action="/proses_login.php" method="POST">
  
</form>
action : Alamat URL/File (biasanya Backend) yang akan memproses data tersebut.
method : Cara pengiriman. GET (data terlihat di URL, untuk pencarian) vs POST (data tersembunyi, untuk password).

03 // Varian Elemen <input>

Tag <input> sangat ajaib. Hanya dengan mengganti atribut type, ia bisa berubah wujud menjadi kotak teks, password, atau tanggal.

<input type="email">

04 // Pentingnya Tag <label>

Jangan pernah meletakkan teks penamaan form tanpa tag <label>. Label menghubungkan teks dengan kotak input, sehingga jika label diklik, kotak input akan otomatis aktif (fokus).

LINKING LABEL & INPUT
<label for="namaDepan">Nama Depan:</label>
<input type="text" id="namaDepan" name="nama">
  • Atribut for pada label harus sama persis dengan nilai atribut id pada input.
  • Atribut name adalah variabel wajib agar server Backend tahu nama data yang dikirim.

05 // Opsi: Radio vs Checkbox

RADIO (Pilih Salah Satu)

Cocok untuk Jenis Kelamin. Wajib menggunakan atribut name yang sama agar saling terkait.

<input type="radio" name="gender"> Pria
<input type="radio" name="gender"> Wanita
CHECKBOX (Pilih Banyak)

Cocok untuk Hobi atau Syarat Ketentuan.

<input type="checkbox"> Coding
<input type="checkbox"> Gaming
Fakta UX: Radio Button dinamakan berdasarkan tombol radio tape mobil lawas yang hanya mengizinkan satu tombol ditekan pada satu waktu.

06 // Menu Dropdown <select>

Jika pilihan terlalu banyak (misal: Daftar Kota/Negara), jangan gunakan Radio Button karena akan menghabiskan ruang layar.

Gunakan kombinasi tag pembungkus <select> dan item list <option> untuk menghemat ruang antarmuka.

<select name="kota">
  <option value="jkt">Jakarta</option>
  <option value="bdg">Bandung</option>
  <option value="sby">Surabaya</option>
</select>

07 // Teks Panjang & Tombol Aksi

<textarea>

Untuk menampung inputan pesan, alamat, atau teks berbaris ganda (multi-line). Berbeda dengan `input`, ia punya tag penutup.

<textarea rows="4"></textarea>
<button>

Triger eksekusi form. Gunakan atribut type="submit" agar seluruh data dikemas dan dikirim ke Action URL.

<button type="submit">KIRIM</button>
